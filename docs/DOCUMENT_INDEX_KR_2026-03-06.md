## 9. 읽는 기준
- `핵심 기준 문서`: 현재 판단의 source of truth
- `배포/운영 문서`: 실제 배포와 운영 의사결정용
- `AI 기능 문서`: 기능 묶음 상태 파악용
- `monetization / billing 문서`: 과금 구조 판단용
- `스펙/경계 문서`: 현재 API 경계 참고용
- `archive / historical`: 단독 기준으로 쓰지 않음

## 10. legacy 배너 원칙
상단에 `legacy/참고용` 배너가 붙은 문서는 아래처럼 취급한다.
- 현재 코드와 1:1로 맞다고 가정하지 않는다.
- source of truth 문서와 같이 읽는다.
- 구현/배포 판단의 단독 기준으로 쓰지 않는다.
